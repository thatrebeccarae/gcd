# Write From Signal

Turn a detected content signal into a published-ready draft by routing through the right agent chain.

## What This Skill Does

Reads content brief stubs generated by the Daily Feed Intelligence n8n workflow, matches a topic, and orchestrates the appropriate agent pipeline based on the brief's `route` field.

## How to Use

```
/write-from-signal [topic]        # Match a topic and start the pipeline
/write-from-signal                # List today's available briefs
```

## Prerequisites

Brief stubs must exist in `~/your-vault/briefs/`. These come from two sources:

- **Pipeline briefs** — Auto-generated daily by the n8n workflow when signals score high enough
- **Manual briefs** — Created via `/seed-idea [title]` for ad-hoc content ideas (identified by `source: manual` in frontmatter)

Both types use the same `route` and `status` fields and are fully compatible with this skill.

## Workflow

### 1. Find the Brief

- Look in `~/your-vault/briefs/` for today's date
- Fallback: yesterday's briefs
- Fallback: read most recent `content-signals/YYYY-MM-DD-signals.md` and offer topic selection for `/research-digest`
- Match `[topic]` argument against filenames and frontmatter (fuzzy match)
- If no topic provided, list all available briefs with scores and routes

### 2. Route by Content Type

Extract `route` from the brief's YAML frontmatter and run the corresponding agent chain.

#### Route: `essay` (Substack long-form)

| Stage | Agent | Model | Output |
|-------|-------|-------|--------|
| 1. Research | research-analyst | opus | `01-Inbox/research-briefs/YYYY-MM-DD-[slug].md` |
| 2. Draft | content-marketer | sonnet | `03-Areas/professional-content/content/YYYY/YYYY-MM-DD-SS-[slug].md` |
| 3. Review | editor-in-chief | opus | Review notes as HTML comments in draft |
| 4. Social | social-amplifier | sonnet | Separate social pack file |

#### Route: `linkedin`

| Stage | Agent | Model | Output |
|-------|-------|-------|--------|
| 1. Draft | content-marketer | sonnet | `03-Areas/professional-content/content/YYYY/YYYY-MM-DD-LI-[slug].md` |
| 2. Review | editor-in-chief | opus | Review notes alongside |
| 3. Social | social-amplifier | sonnet | Twitter cross-post + email subjects |

#### Route: `twitter-thread`

| Stage | Agent | Model | Output |
|-------|-------|-------|--------|
| 1. Draft | content-marketer | sonnet | `03-Areas/professional-content/content/YYYY/YYYY-MM-DD-TW-[slug].md` |
| 2. Social | social-amplifier | sonnet | LinkedIn cross-post variant |

### 3. User Gates

After each stage, pause and show:
- Output file location
- Brief summary of what was produced
- Options: **Continue** to next stage / **Give feedback** for revision / **Stop** here

### 4. Status Tracking

Update the brief stub's frontmatter `status` field:
- `stub` -> `in-progress` (when pipeline starts)
- `in-progress` -> `published` (when user confirms final output)

## Context Passing

- Brief content is passed inline to the first agent (small, <500 words)
- Subsequent agents receive file paths to read (they have Read tool access)

## Examples

```
/write-from-signal ai agents           # Fuzzy matches a brief about AI agents
/write-from-signal                      # Lists all available briefs
```
